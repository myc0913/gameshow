# A0 阶段设计文档：项目骨架与静态页面

## 概述

初始化 Vite + React + TypeScript 项目，搭建 3 个 Tab 页面框架，创建 Play 页的空白占位区。不写任何规则引擎逻辑。

## 技术选型

| 项 | 决定 |
|----|------|
| 脚手架 | Vite + React 18 + TypeScript 5 |
| CSS 方案 | 全局 `styles.css` 做底色和布局；后续组件阶段引入 CSS Modules |
| 状态管理 | 本阶段无状态，纯静态占位 |
| Three.js | 本阶段不引入，动画区仅放 `<canvas>` 占位框 |

## 页面结构

```
App
 ├── TabNav (Play | How | Why)
 └── 当前页内容
      ├── PlayPage  (默认显示)
      ├── HowPage
      └── WhyPage
```

路由用简单 state 切换，不引入 react-router（A0 不需要）。

## Play 页布局：左右分栏

```
┌────────────────────────────────────────────┐
│  🏷️  Play  |  How  |  Why                  │
├────────────────────┬───────────────────────┤
│                    │                       │
│  🎒 符文池         │  📋 技能结果面板      │
│  (符文卡片网格)    │  (空态占位)           │
│                    │                       │
│  📥 符文槽位栏     │  🎬 技能动画区        │
│  (4 槽位水平排列)  │  (Canvas 占位框)      │
│                    │                       │
│  ⚙️ [铭刻] 按钮   │                       │
│                    │                       │
└────────────────────┴───────────────────────┘
```

- 左栏（40%）：符文池 + 槽位栏 + 铭刻按钮，纵向排列
- 右栏（60%）：结果面板 + 动画区占位，纵向排列
- 响应式：窄屏时切换为纵向堆叠

## 目录结构

```
src/
  data/           — 本阶段留空（A1 加符文数据）
  engine/         — 本阶段仅建目录（A1 加纯函数）
  components/     — TabNav（本阶段唯一有逻辑的组件）
  pages/          — PlayPage / HowPage / WhyPage（静态占位）
  types/          — rune.ts / skill.ts（本阶段定义类型骨架）
  App.tsx         — Tab 切换 state
  main.tsx        — 入口
  styles.css      — 全局样式
```

## 验收标准

1. `npm run dev` 可启动
2. 默认进入 Play 页
3. Play / How / Why 三个 Tab 可切换
4. 控制台无报错
5. Play 页可见：符文池空区、4 槽位占位、铭刻按钮、结果面板空态、动画区 Canvas 框
